## Step 1: Create the parameter file

<br>

*In Linux:*

`sudo docker run -it -v /home/GIS/force:/data davidfrantz/force force-parameter /data/parameter_ML_RFC.prm ML`

`[sudo] password for birdlab:` 

Output:
```
An empty parameter file skeleton was written to
  /data/parameter_ML_RFC.prm
Note that all parameters need to be given, even though some may not be used
with your specific parameterization.
Parameterize according to your needs and run with
force-higher-level /data/parameter_ML_RFC.prm
```
<br>

*In Windows:*

`podman run -it -v "C:\Users\GIS:/home/docker/force" davidfrantz/force force-parameter /home/docker/force/force/parameter_ML_RFC.prm ML`

Output:
```
Similar to Linux
```

## Step 2: Make the necessary directories

<br>

*In Linux:*

Create the output directory:<br>
`sudo docker run -it -v /home/GIS/force:/data davidfrantz/force mkdir /data/dir_ml_RFC`

Create the provenance file directory (only is needed for any number of models, as long as the models are to be outputted in the same main directory like 'force'):<br>
`sudo docker run -it -v /home/GIS/force:/data davidfrantz/force mkdir /data/dir_provenance_ml`

<br>

*In Windows:*

Create the output directory:<br>
`podman run -it --user root -v "C:\Users\GIS:/home/docker/force" davidfrantz/force mkdir /home/docker/force/force/dir_ml_RFC`

Create the provenance file directory (only is needed for any number of models, as long as the models are to be outputted in the same main directory like 'force'):<br>
`podman run -it --user root -v "C:\Users\GIS:/home/docker/force" davidfrantz/force mkdir /home/docker/force/force/dir_provenance_ml`

## Step 4: Run the program

<br>

*In Linux:*


`sudo docker run -it -v /home/GIS/force:/data davidfrantz/force force-higher-level /data/parameter_ML_RFC.prm`

*In Windows:*

`podman run -it -v "C:\Users\GIS:/home/docker/force" davidfrantz/force force-higher-level /home/docker/force/force/parameter_ML_RFC.prm`

If there are strip gaps in the output which can be found in the directory "dir_ml_RFC", rerun for the tiles which needs to be processed (due to missing chunks) with the use of tile allow-lists, until all are fixed. It may take multiple attempts.
